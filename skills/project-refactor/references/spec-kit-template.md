# SPECKIT 过程文件模板

商业净室重建在执行契约确认后创建 `specs/NNN-功能名/`。这些文件是恢复任务和审查交付的唯一过程事实来源；动态状态不能只留在对话中。

## 根入口文件

在 `AGENTS.md` 和 `CLAUDE.md` 写入内容相同的最小区块。保留区块外既有内容；只更新该区块。

```md
<!-- SPECKIT START -->
开始、恢复、变更范围或发布当前重构任务前，必须读取：
`specs/NNN-功能名/plan.md`、`tasks.md`、`ledger.md`、`commercial.md` 和 `handoff.md`，再核对 Git 状态与基线。

当前活跃 feature：[specs/NNN-功能名/plan.md](specs/NNN-功能名/plan.md) — <一句话状态>。

硬边界：<净室输入限制、不可触碰模块、商业发布阻断项>。
恢复规则：未完成上述读取和核对，不得修改代码或标记阶段完成；完成可验证阶段后必须更新过程文件与本区块。
<!-- SPECKIT END -->
```
